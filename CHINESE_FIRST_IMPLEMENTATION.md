# 课堂演示优化 - 中文优先多语言策略实现

**更新日期**: 2026-03-21  
**核心改动**: 将所有 benchmark 案例和示例改为**中文优先**，支持多语言对照（波斯语、阿拉伯语、英文等）

---

## 📋 完整改动清单

### 1. 核心数据层改动

#### 📄 `common/benchmark.json` - 扩展评测任务库
**原状态**: 仅包含 stage1 的 3 个简单任务
**新增任务**:
- ✅ **XNLI** - 跨语言自然语言推理 (3 个案例)
  - 案例 1: 否定陷阱 (中文→波斯→阿拉伯→英文)
  - 案例 2: 数字陷阱 (中文→波斯→阿拉伯→英文)
  - 案例 3: 中性判断 (中文→波斯→阿拉伯→英文)

- ✅ **MGSM** - 多语言小学数学题 (4 个案例)
  - 案例 1-4: 加法、复杂减加、乘法、时间计算
  - 每个都支持: 中文→波斯→英文→阿拉伯(→印地语)

- ✅ **FLORES-200** - 翻译质量评测 (4 个案例)
  - **中文为源语言** (重点!)
  - 翻译目标: 英文、波斯语、德文、西班牙语、法文

- ✅ **XQuAD** - 跨语言问答 (3 个案例)
  - 时间抽取、地点抽取、人物识别
  - 多语言版本: 中文→波斯→英文

- ✅ **BELEBELE** - 阅读理解 (2 个案例)
  - 简单理解、推理理解
  - 多语言版本: 中文→波斯→英文

#### 📄 `common/utils.py` - 重构 EXAMPLE_INPUTS
**原策略**: 混乱的语言顺序，英文常常优先
**新策略**: **严格中文优先**，配合多语言对照

**改动详情**:
```python
# 每个模型的示例现在遵循：
[
  {"lang": "中文", "text": "..."},     # 第一位：中文（主展示）
  {"lang": "波斯语", "text": "..."},    # 第二位：波斯语（重点对照）
  {"lang": "阿拉伯语", "text": "..."},  # 第三位：阿拉伯语
  {"lang": "英文", "text": "..."},      # 第四位：英文（国际标准）
  # ... 其他语言
]
```

**示例重构** (不完全列表):
| 模型 | 原策略 | 新策略 |
|-----|-------|--------|
| word2vec | 中文/英文/西班牙语 | 中文优先，加波斯语、阿拉伯语 |
| bert | 全英文 | 中文优先示例 |
| mbert | 混合 | 中文→波斯→英文→西班牙 |
| xlm | 中英平行只有一个 | 中英、中+波斯多个对比 |
| bloom | 纯英文/法文 | 中文生成示例优先 |

## 🎯 具体演示场景

### 场景 1: 课堂讲解 XNLI 跨语言推理
1. **打开** `benchmark.py` 页面
2. **选择** XNLI 任务 (左侧 Sidebar)
3. **查看** 案例 1（否定陷阱）
   - 默认显示中文版本
   - 学生快速理解题意（"那个男孩在踢足球" vs "没有在踢足球"）
4. **切换**语言到波斯语
   - 观察文字从左→右变成右→左
   - 预测: 模型是否能处理这种差异？
5. **运行模型**，查看实际结果
6. **对比**: 同一道题，不同语言，模型表现是否相同？

### 场景 2: 演示 Stage 2 跨语言迁移
1. **打开** `stage2.py` 页面
2. 先看语言设计说明，理解"零样本跨语言迁移"含义
3. **展开** mBERT 模型
   - 看到多语言示例 Tab（中文优先）
4. **运行**中文填空: "中国的首都是[MASK]。"
   - 看到结果（应该是：北京）
5. **修改**输入为波斯语: "پایتخت چین [MASK] است."
   - 预测: 会不会也成功？
6. **运行**，对比结果
7. **讨论**: 为什么不同语言效果相差大/相差小？

### 场景 3: MGSM 数学推理多语言
1. **打开** `benchmark.py`
2. **选择** MGSM 任务
3. **查看** 案例 1（简单加法）
   - 中文: "小明有3个苹果，又买了2个..."
   - "这道题很简单，答案是5"
4. **切换**到波斯语版本
   - "同样的逻辑，只是换了语言"
5. **讨论**: 模型在波斯语中能否做到同样的推理？

## 🎨 语言多样性设计

### 为什么这些语言？

| 语言 | 优先级 | 原因 | 位置 |
|-----|-------|------|------|
| 中文 | ⭐⭐⭐⭐⭐ | 课堂受众、高资源、CJK特性 | **第 1 位（默认）** |
| 波斯语 | ⭐⭐⭐⭐ | RTL文字、文化差异、视觉冲击 | **第 2 位** |
| 阿拉伯语 | ⭐⭐⭐⭐ | RTL + 多使用人口、低资源 | **第 3 位** |
| 英文 | ⭐⭐⭐ | 国际标准、论文对标 | **第 4 位** |
| 印地语 | ⭐⭐⭐ | 印度最大语言、低资源、非拉丁 | **第 5 位** |
| 法文/西班牙 | ⭐⭐ | 补充高资源语言 | **按需加** |

### 关键设计理念

```
❌ 坏的设计:
   - "这个模型支持100种语言！"（然后全用英文演示）
   - 只展示高资源语言（英文、中文），无法证明真正的多语言能力

✅ 好的设计（本项目):
   - 同一道题 → 中文、波斯语、阿拉伯语 → 看表现是否接近
   - 如果模型说自己"多语言"，那波斯语也应该工作
   - 如果波斯语突然崩溃，说明"多语言"其实只是"多个语言的LM"
```

## 🚀 使用建议

### 💼 课堂演示最佳实践

1. **快速理解** (2 分钟)
   - 用中文示例快速解释题意
   - 学生无需翻译，节省时间

2. **多语言对比** (3 分钟)
   - 切换到波斯语/阿拉伯语
   - "同样的逻辑，不同的文字"
   - 预测：模型能否处理？

3. **实验验证** (3 分钟)
   - 输入模型，看实际结果
   - 对比预期 vs 实际
   - "哪里符合预期？哪里超出预期？"

4. **总结规律** (2 分钟)
   - "模型在中文表现好，为什么波斯语不好？"
   - "这说明什么关于多语言模型？"
   - "我们应该如何评估一个真正的多语言模型？"
